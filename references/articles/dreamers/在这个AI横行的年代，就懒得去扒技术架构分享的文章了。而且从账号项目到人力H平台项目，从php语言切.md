代码哲学：写给人类看的机器指令
1. **命名防污术：**
`dto`、`vo` 这类缩写早该进博物馆了！AI时代更要警惕“拼音英文杂交变量”（比如 `shangpinList` 和 `productList` 混用）。我的强迫症方案：
- **领域词中英统一**（要么全用`Order`，要么全用`Dingdan`）
- **动词用英文**（`calculateTax()` 比 `jisuanShui()` 更通用）
- **神器加持**：让Copilot生成5个候选名，挑最像人话的那个

2. **注释预加载模式：**
反对“写完代码补注释”的自我欺骗！我习惯先敲注释框架：
```go
// 目标：把用户订单拆分为可并行处理的子任务
// 关键陷阱：子任务依赖库存锁，需防止死锁
// 逃生出口：超时回滚机制
func splitOrder(order Order) []SubTask {
// ... 后面再填血肉
}
```
**写注释的过程就是设计推演**，能提前发现逻辑漏洞。

3. **代码防脱发守则：**
- **拒做“千行侠”**：单个函数超一屏就警铃大作，拆！
- **警惕炫技陷阱**：Go里硬写PHP式的try catch？省省吧，`if err != nil` 朴实但救命
- **可读性即正义**：代码是给人维护的，不是给机器诵经的。团队里新人能三天看懂代码库，才是真优雅

4. **画图拯救沟通：**
ProcessOn是我的人脑外接显卡。手先画三件套：
🔹 **业务流程图**（泳道图揪出责任真空）
🔹 **存储关系图**（ER图暴露JOIN地狱）
🔹 **时序图**（并发冲突一目了然）
**省下80%的扯皮会议，比什么技术选型都值**

代码那点事儿：让人看得懂是头等大事

1.  命名别作妖：
`dto`、`vo` 这些缩写，新人看了懵圈，自己过仨月也未必记得！最怕中英混搭的妖怪名（`userList` 和 `yonghuList` 齐飞）。我的土规矩：
同一个意思，要么全英文，要么全拼音（订单就叫 `Order` 或者 `Dingdan`，别混着来）。我还是建议采用英文缩写或者拼写更加通用一点。哪怕是拼错了单词。
动作用英文`calculatePrice()` 肯定比 `jisuanJiage()` 顺眼）。
善用AI提词器：Copilot/ChatGPT 生成几个候选名，挑个最像人说的人话。

2.  注释先打草稿：
```go
// 干啥用的：批量确认离职把一个大循环分成多个go协程
// 坑在哪：第三方打挂！
// 留后路优化点：todo异步！
```
写着写着注释，在AI的辅助下，代码都直接给跳出来了。

3.  代码清爽点，别恶心人：
拒写“裹脚布”函数：一个函数长得要死，滚鼠标滚轮都滚不到底？拆！必须拆！
别硬凹造型：用Go就好好写Go，别整PHP那套 `try...catch` 硬塞进来，老老实实 `if err != nil` 检查错误，虽然啰嗦但踏实。
可读性是金标准：代码是给人看的！新来的兄弟三天内能摸清咱这摊代码是干啥的、怎么干的，那才叫好代码！

4.  画图！
ProcessOn（或者你喜欢的画图工具）真是神器！接到新需求，先画它三张图：
业务流程图（辅助自己跟他人快速明白业务~）
数据库关系图（表怎么连，JOIN会不会搞死人？一目了然）
时序图（几个东西一起动，会不会打架？画出来就清楚）

## ⌨️ 代码那点事儿：让人看得懂是头等大事

### 1️⃣ 命名别作妖
- **`dto`、`vo` 这些缩写，新人看了懵圈，自己过仨月也未必记得！**
- **最怕中英混搭的妖怪名（`userList` 和 `yonghuList` 齐飞）**

**我的土规矩：**
- **同一个意思，要么全英文，要么全拼音**（订单就叫 `Order` 或者 `Dingdan`，别混着来）
- **动作用英文`calculatePrice()` 肯定比 `jisuanJiage()` 顺眼**
- **善用AI提词器：Copilot/ChatGPT 生成几个候选名，挑个最像人说的人话**

### 2️⃣ 注释先打草稿
```go
// 干啥用的：批量确认离职把一个大循环分成多个go协程
// 坑在哪：第三方打挂！
// 留后路优化点：todo异步！
```
> **写着写着注释，在AI的辅助下，代码都直接给跳出来了**

### 3️⃣ 代码清爽点，别恶心人
- **拒写"裹脚布"函数**：一个函数长得要死，滚鼠标滚轮都滚不到底？**拆！必须拆！**
- **别硬凹造型**：用Go就好好写Go，别整PHP那套 `try...catch` 硬塞进来，**老老实实 `if err != nil` 检查错误**，虽然啰嗦但踏实
- **可读性是金标准**：代码是给人看的！**新来的兄弟三天内能摸清咱这摊代码是干啥的、怎么干的，那才叫好代码！**

### 4️⃣ 画图！
> **ProcessOn（或者你喜欢的画图工具）真是神器！接到新需求，先画它三张图：**

- **业务流程图**（辅助自己跟他人快速明白业务~）
- **数据库关系图**（表怎么连，JOIN会不会搞死人？一目了然）
- **时序图**（几个东西一起动，会不会打架？画出来就清楚）
