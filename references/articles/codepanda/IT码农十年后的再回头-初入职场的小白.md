IT码农十年后的再回头-初入职场的小白

作为一名在IT行业，积累了十多年的工作经历的码农。再来回头看看当初那个初入职场的小白，依旧感慨万千。

如今作为一名过来人，会时常反思这些年走过路一些得失。时常会想，如果当时做出不同的抉择，如今的结局会比现在好很多。有时候也会安慰自己，这些中间弯路也好，坦途也罢，都是自己心智跟年纪成熟的催熟剂。

某友科技，是我真正职场开端的地方。这虽然是一家很渣的公司，项目很渣、技术架构很烂，但是不可否认的，它的宽松的环境以及当时的leader，还是给了我足够的耐心，给我来慢慢适应职场。因为技术架构很烂，因为都是最原生的php代码结合原生的MySQL。所以里面涉及的安全性很低，代码效率很低、可维护性很差，慢查询满天飞~ 但是这样确实可以让一个新人小白快速把php跟MySQL技术，理论跟实操结合起来的项目。同时因为这种原生的项目，会催生自己的思考。如何写出高效率的代码跟如何避免这种低效，难以维护的项目代码。

自己思考之后的总结的经验：
1、代码的健壮性跟安全性，一定要考虑任何用户的输入的非法性。不能信任任何用户的输入，以及一定要考虑各种边界条件。 对各种异常情况的考虑跟兼容处理，是我觉得衡量一个好坏程序员的分界线。【php的try catach很重要，不要让exception乱飞~跟不可控。】
2、对于注释、变量方法命名的有意义并且风格统一很重要。【不管是驼峰，还是中文拼音命名，关键是一定要风格统一且命名有意义。代码注释很重要。】
3、不要循环里面IO，尽量少的IO操作，把这种循环里面IO操作，尽量改造成一次IO，把逻辑用代码实现。【最常见的例子，循环sql查询~】
4、sql语句一定要考虑效率，结合实际项目，建立合适的索引，解决cover大部分业务场景。不要产生慢查询的情况跟非安全情况。尽量在追求效率的业务接口不要用复杂的子查询，嵌套子查询，联合查询这种复杂的SQL语句。【索引可以解决90%的问题，合理的使用SQL语句，可以解决95%的问题】
5、一定要合理设计库表模型。库表模型要灵活，尽量考虑后续的需求变更以便可以最少变动适配。【MySQL建立库表，不需要追求范式的理论，可以结合实际业务情况，做适当的字段冗余】

关于职场的经验总结：
1、技术实力是作为打工人在公司赖以生存的根本。所以职场小白跟公司谈待遇的时候，技术实力是你的底气。【我在经历了10个月的打怪升级之后，果断要求公司给我涨薪。因为我下一家已经找好了。如果公司给我涨薪的条件不能达到我的预期，我就直接跳槽走了。】
2、要自信，要敢于去表现，敢于去争取。虽然从学生到打工人身份的转变，可能对自身的未来不太明朗。尤其是在求职过程遭受各种挫折跟打击，但是一定要丧失掉自己的自信。一定要自信。哪怕自己是小白，也不要害怕公司或者HR看不上你，从而不敢表现自己，不敢谈薪资。我们可以正常表达自己，作为初入职场的菜鸟，公司看中的是你的学习能力跟长远你的价值，而不是你现在能够提供的价值。所以认清自己很重要，但是一定不要看轻自己。【我自己的职场开场是一个天坑，自己事后复盘，如果当场自己开局如果好一点，后面的结果应该会好很多~】
3、跟同事搞好关系，是出入职场新人的必修课。因为刚入社会，你的同事可能是在这个陌生城市，唯一可以一起吃饭，聊天，玩乐的伙伴。尤其是作为职场新人面临租房子，周末无聊时光等问题。有几个聊得来的同事，是很重要的一件事。

IT码农十年后的再回头 - 初入职场的小白
作为一名在IT行业积累了十多年工作经验的码农，再次回顾当初那个初入职场的小白，心中依旧感慨万千。

如今，身为过来人，我常常反思这些年来走过的路，总结得失。有时会想，如果当时做出了不同的选择，如今的结局是否会更好。然而，也常自我安慰，这些弯路也好，坦途也罢，都是促使自己心智和年龄成熟的催化剂。

某友科技，是我真正开启职业生涯的地方。尽管这是一家很渣的公司，项目糟糕、技术架构混乱，但不可否认，它宽松的环境以及当时的领导给了我足够的耐心，让我慢慢适应职场。由于技术架构原始，采用最基础的PHP代码结合原生的MySQL，因此安全性低下，代码效率低，可维护性差，慢查询频繁。但正是这样一个烂摊子，却让我这个新人快速将PHP与MySQL技术理论与实操结合起来。同时，这种简单粗暴的项目也激发了我的思考：如何编写高效代码，如何避免低效且难以维护的项目代码？

经过自己的思考总结，我得出以下经验：

代码的健壮性和安全性：一定要考虑用户输入的非法性，不能信任任何用户输入，并且要处理各种边界条件。对异常情况的兼容处理，是衡量程序员水平的重要标准。【PHP 的 try-catch 很重要，不要让 exception 随意飞散而不可控。】

有意义且统一风格的注释和命名：无论使用驼峰命名法还是中文拼音命名，关键是风格统一且命名有意义。代码注释也非常重要。

减少循环中的 I/O 操作：尽量避免在循环中进行 I/O 操作，将其改造成一次性的 I/O 操作，用代码实现逻辑。【例如，避免循环中的 SQL 查询。】

优化 SQL 语句：考虑 SQL 效率，根据实际项目建立合适的索引，解决大部分业务需求。避免复杂的子查询、嵌套查询和联合查询。【合理使用索引可以解决 90% 的问题，而高效的 SQL 语句可以解决 95% 的问题。】

合理设计数据库表模型：库表模型要灵活，预见后续需求变更，以便最少变动就能适配。【不必追求范式理论，可以根据实际业务情况适当冗余字段。】

关于职场的经验总结：

技术实力是立足之本：技术实力是打工人在公司赖以生存的根本。谈待遇时，技术实力是你的底气。【在经历了10个月的成长之后，我果断要求公司涨薪，因为下一家公司已经找好了。如果达不到预期，我就跳槽离开。】

自信并敢于表现：从学生到打工人的身份转变可能令人迷茫，尤其是在求职过程中遭遇挫折，但绝不要丧失自信。即使是小白，也不要害怕被公司或 HR 看不上，从而不敢表现自己、不敢谈薪资。公司看重的是你的学习能力和长远价值，而不是你现在能够提供的价值。所以，要认清自己，但绝不要看轻自己。【我的职场起点虽然坎坷，但事后复盘，如果开局好一些，结果应该会好很多。】

搞好同事关系：这是职场新人的必修课。同事可能是你在陌生城市唯一可以一起吃饭、聊天、玩乐的伙伴。特别是面对租房、周末无聊等问题，有几个聊得来的同事至关重要。

希望这些经验教训，能为初入职场的你们带来一些启示和帮助。每一步路，无论顺遂还是曲折，都是我们成长的一部分。
