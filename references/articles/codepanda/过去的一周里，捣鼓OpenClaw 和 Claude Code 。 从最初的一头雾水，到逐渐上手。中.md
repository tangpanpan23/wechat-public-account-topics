# 折腾 OpenClaw + Claude Code 三天后，我的一些思考和心得

工具本身不产生价值，使用工具的人才能创造价值。

---

过去三天，我几乎把所有业余时间都花在了折腾 OpenClaw 和 Claude Code 上。
从最初的一头雾水，到逐渐上手，再到现在的游刃有余。
中间踩过坑、走过弯路、也收获了不少惊喜。
今天想把这些经验跟心得记录下来，给想尝试的朋友一些参考。
也给自己一个复盘的机会。

---

## 01 — 为什么要折腾这个？

先说动机。

我一直对 AI 工具保持着一种既期待又警惕的态度。
期待的是，它确实能提升效率，帮我解决一些重复性工作。
警惕的是，我怕过度依赖，怕丧失独立思考的能力。

这次折腾 OpenClaw + Claude Code，其实是想验证一件事：
AI 工具到底能在多大程度上，真正帮助到一个普通开发者？
是噱头大于实用，还是真的能改变工作方式？

三天下来，我有一些答案了。

---

## 02 — OpenClaw 是什么？

先简单科普一下。

OpenClaw 是一个开源的 AI 助手框架，可以理解为“AI 的 AI”。
它的核心能力是：
- 统一管理多个 AI 模型（Claude、GPT、Gemini 等）
- 自动化工作流（定时任务、消息推送、数据处理）
- 扩展能力强（可以集成飞书、微信、GitHub 等）
- 本地部署（数据在自己手里，安全可控）

我这次主要用它来做什么？
- 自动抓取热点，生成公众号选题
- 分析历史文章风格，辅助写作
- 管理飞书消息，自动通知
- 定时任务，比如每天推送选题

简单说，就是让它帮我打理一些重复性的工作，让我有更多时间做更有价值的事。

---

## 03 — Claude Code 又是什么？

Claude Code 是 Anthropic 推出的编程助手，基于 Claude 模型。
跟我之前用过的 Copilot、Cursor 相比，它有以下几个特点：

**优点：**
1. 理解能力强 – 能理解复杂的项目结构和业务逻辑
2. 代码质量高 – 生成的代码规范、可读性好
3. 对话流畅 – 可以多轮对话，逐步完善需求
4. 支持长上下文 – 能处理大型项目，不会丢三落四

**缺点：**
1. 需要科学上网 – 这是硬伤
2. 付费 – 好工具都不便宜
3. 偶尔幻觉 – 生成的代码需要人工 review

总体来说，Claude Code 是我目前用过最顺手的编程助手。

---

## 04 — 三天折腾历程

**第一天：踩坑日**
第一天基本在踩坑。
- 环境配置问题（Node 版本、依赖冲突）
- API 密钥配置（飞书、微信公众号）
- 权限问题（OAuth 授权、IP 白名单）
- 各种报错（看不懂、搜不到、问 AI 也答非所问）

说实话，第一天结束的时候，我有点怀疑人生了。
这玩意儿真的适合普通人用吗？

但第二天，事情开始有转机。

**第二天：开悟日**
第二天，我改变了策略。
不再追求“一次性搞定所有功能”，而是：
1. 先跑通最小工作流 – 能发一条飞书消息就行
2. 逐步添加功能 – 每添加一个功能，确保它能正常工作
3. 记录问题 – 遇到坑就记下来，方便后续复盘

这个方法很有效。
到第二天晚上，我已经能实现：
- 自动抓取热点
- 生成选题
- 飞书推送通知

虽然还不完美，但基本流程跑通了。

**第三天：收获日**
第三天，我开始优化和扩展。
- 修复了 HTML 格式问题（微信公众号草稿保存）
- 优化了飞书通知格式（机器人模式）
- 添加了定时任务（每天自动推送选题）
- 写了一些自定义脚本（风格分析、文章生成）

到第三天结束，整个系统基本成型。
我可以自豪地说：
我有了一个属于自己的 AI 助手系统。

---

## 05 — 一些经验和心得

1. **不要追求完美，先跑起来**
很多人（包括我）容易陷入一个误区：想要一次性把所有功能都配置好，追求完美。结果就是，配置了半天，什么都没跑通，最后放弃了。
我的建议是：先跑通最小工作流，再逐步完善。比如 OpenClaw，先让它能发一条消息，再考虑其他功能。

2. **善用 AI 助手，但别完全依赖**
这三天，我几乎每个问题都问了 Claude Code。但我也发现，完全依赖 AI 是有问题的：
- AI 会幻觉，给出的答案不一定对
- AI 不理解你的具体场景，建议可能不适用
- AI 无法替代你的思考，最终决策还得你自己来
我的做法是：把 AI 当助手，不当老师。它给出的建议，我会思考、验证、再决定是否采纳。

3. **记录问题，方便复盘**
这三天我踩了很多坑，也解决了很多问题。我习惯把每个问题和解决方案都记下来。这样做的好处是：
- 下次遇到同样问题，可以快速解决
- 复盘的时候，能看到自己的成长轨迹
- 分享给别人，也能帮助别人避坑

4. **工具是手段，不是目的**
这是我最想强调的一点。折腾工具本身不产生价值。使用工具解决问题，才能产生价值。
我折腾 OpenClaw + Claude Code，不是为了炫耀技术，而是为了：
- 提升写作效率
- 减少重复劳动
- 有更多时间做更有价值的事
如果工具不能帮你实现这些目标，那它就只是一个玩具。

## 06 — 关于 AI 工具的思考

这三天，我对 AI 工具有了一些更深的思考。

AI 不会取代人，但会取代不会用 AI 的人。
这句话听起来有点鸡汤，但确实是现实。

我见过太多人，对 AI 工具持两种极端态度：
- 一种是盲目崇拜，觉得 AI 能解决一切问题
- 一种是完全排斥，觉得 AI 就是噱头

我觉得这两种态度都不可取。
正确的态度应该是：保持开放，但保持独立思考。
开放地尝试新工具，但思考它是否真的适合你。
