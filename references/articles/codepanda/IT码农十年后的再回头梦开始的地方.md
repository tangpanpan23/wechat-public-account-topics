码农生涯的开端

作为一名在IT行业积累了十多年工作经验的码农，再次回顾当初那个初入职场的小白，心中依旧感慨万千。

如今，身为过来人，我常常反思这些年来走过的路，总结得失。

有时会想，如果当时做出了不同的选择，如今的结局是否会更好。

然而，也常自我安慰，这些弯路也好，坦途也罢，都是促使自己心智和年龄成熟的催化剂。

某友科技，是我真正开启职业生涯的地方。尽管这是一家很渣的公司，项目糟糕、技术架构混乱，但不可否认，它宽松的环境以及当时的领导给了我足够的耐心，让我慢慢适应职场。

由于技术架构原始，采用最基础的PHP代码结合原生的MySQL，因此安全性低下，代码效率低，可维护性差，慢查询频繁。

但正是这样一个烂摊子，却让我这个新人快速将PHP与MySQL技术理论与实操结合起来。

同时，这种简单粗暴的项目也激发了我的思考：如何编写高效代码，如何避免低效且难以维护的项目代码？

小白职场感悟

经过自己一年的工作经历的思考总结，得出以下经验：

**代码的健壮性和安全性** ：

一定要考虑用户输入的非法性，不能信任任何用户输入，并且要处理各种边界条件。对异常情况的兼容处理，是衡量程序员水平的重要标准。【PHP 的 try-catch
很重要，不要让 exception 随意飞散而不可控。】

**有意义且统一风格的注释和命名** ：

无论使用驼峰命名法还是中文拼音命名，关键是风格统一且命名有意义。代码注释也非常重要。

**减少循环中的 I/O 操作** ：

尽量避免在循环中进行 I/O 操作，将其改造成一次性的 I/O 操作，用代码实现逻辑。【例如，避免循环中的 SQL 查询。】

**优化 SQL 语句** ：

考虑 SQL 效率，根据实际项目建立合适的索引，解决大部分业务需求。避免复杂的子查询、嵌套查询和联合查询。【合理使用索引可以解决 90% 的问题，而高效的
SQL 语句可以解决 95% 的问题。】

**合理设计数据库表模型** ：

库表模型要灵活，预见后续需求变更，以便最少变动就能适配。【不必追求范式理论，可以根据实际业务情况适当冗余字段。】

菜鸟职场总结

**技术实力是立足之本** ：

技术实力是打工人在公司赖以生存的根本。谈待遇时，技术实力是你的底气。【在经历了10个月的成长之后，我果断要求公司涨薪，因为下一家公司已经找好了。如果达不到预期，我就跳槽离开。】

**自信并敢于表现** ：

从学生到打工人的身份转变可能令人迷茫，尤其是在求职过程中遭遇挫折，但绝不要丧失自信。

即使是小白，也不要害怕被公司或 HR 看不上，从而不敢表现自己、不敢谈薪资。

公司看重的是你的学习能力和长远价值，而不是你现在能够提供的价值。所以，要认清自己，但绝不要看轻自己。【我的职场起点虽然坎坷，但事后复盘，如果开局好一些，结果应该会好很多。】

**搞好同事关系** ：

这是职场新人的必修课。同事可能是你在陌生城市唯一可以一起吃饭、聊天、玩乐的伙伴。特别是面对租房、周末无聊等问题，有几个聊得来的同事至关重要。

![](https://mmbiz.qpic.cn/mmbiz_jpg/WAcdGrGyFXDOWHf4arq4QJVIiccAbibYvTfrRZ4gWA9p0w6cpvcguUyrN9tjmUmib6tCsibGBtMueGxmKFQkyXUXlA/640?wx_fmt=jpeg&from=appmsg)

希望作为过来在职场的这些经验教训，能为初入职场的菜鸟，当初跟我一样迷茫的新人带来一些启示和帮助。每一步路，无论顺遂还是曲折，都是成长的一部分。也希望从学校步入职场的每个新人，都有一个美好未来。
