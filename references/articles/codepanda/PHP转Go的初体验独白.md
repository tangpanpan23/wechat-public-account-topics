#  PHP转Go的初体验独白

原创  功夫熊猫盼  功夫熊猫盼  [ 码农世界CodePanda ](javascript:void\(0\);)

_2022年03月10日 11:44_ __ _ _ _ _

在流性使然的前提下，为了保证不被IT互联网时代给劝退。所以决定踏入比PHP更年轻的Go语言这条路，也不知道最后归宿在哪，但是决心好好走走。

学习一门新语言的历程，就个人学习阶段分享一下。不是最佳实践，只是个人觉得比较适合自己这种老菜鸟而已。就跟把大象关进冰箱需要几步一样，第一步，我是先了解一下什么是大象。

首先，花了三天时间，多看看各种官方文档，以及Go社区对go的评价以及设计初衷。对比一下go跟其他语言的优劣势是什么。从各种文档总结出来的结论：go是一门后端语言，是一种更接近C语言的升级版的高级现代语言。而不是市面流行的脚本php、Python这种解释型语言。同时一种比Java更轻便，更直白的编译型的语言。Go的设计初衷，就是简洁，可靠，更高性能的语言，同时兼顾开发效率。一门开箱即用的语言。

然后就是自然安装本地环境，自己入门hello
world。参考官方文档，在Mac安装go真的是易如反掌。Mac本的brew工具，简直是安装各种软件的神器。同时码农写啥语言都逃不开的IDE也必须备齐，GoLand也是必备神器。这里刚开始会对GoRoot,GoPath有点晕~反正依葫芦画瓢就行。不必深究。

中间没啥好说的，参照文档，把所有的语法都过一遍。这里每一种语言都差不多的，变量类型、循环、计算、内置函数、自定义函数~
然后这里基本可以花上三天时间，把所有语法过一遍，也都动手走一遍demo，把go build、go run 调试都走起来~
这里go变量类型较于PHP来说其实没有对比意义，PHP是一种弱语言类型，可以不用申明变量，在执行过程中会自动进行转换。而go是一种类比于C语言的强语言类型，所有变量必须要先声明，再严格按照申明使用。同时循环也是极简模式，for
rang循环一切~ 同时有一种需要注意的变量类型是配合go的设计特点诞生的channel类型。同时if
是简单的连括号范围都省去，同时也省去PHP语言所需要的强制结束标记符。go语言没有显示结束标记。同时go的switch case
是默认break的，不会像其他语言那样默认fallthrough.

最后就是拿公司项目练手，上手写go项目。顺便熟悉一下市面上较为流行的go后端框架Gin。在照猫画虎的前提下，写go项目。所有的业务逻辑都离不开增删改查，最后落地数据库。中间会需要中间件来校验调用的安全性，同时需要脚本来处理一些定时任务获取数据的能力。

总体感觉，go是入门较简单的，门槛比较低的语言。但是想要精通还是需要花功夫的语言。对于停留在写业务逻辑代码，三个月的时候，来写代码是勉强够用的。但是要写出好的代码，同时对底层实现逻辑有一个如同php那种把控，还是需要花时间。最后就是要去阅读源码实现了，go是一门自举的语言，go语言本身就是拿go来实现的。可以参考Gin这种框架的源码，来学习行业优秀的代码。

就如同《中国合伙人》里面说的，学习外语最好的方法，就是用外国人的思维去思考。用go语言的思维，去写go代码。

go标准库，优秀的开源项目~符合go惯用法的高质量项目，是有效进阶的方法途径。

前面都是废话，下面是这三个月沉淀下来的具体感受：

1、go的设计原则，重视错误处理。所有错误都显示处理。go的错误error跟异常panic是分开的。拒绝一切模棱两可的逻辑。

go里面，对fatal、error处理，都是显示，也是go的设计逻辑之一，所以panic需要特别引起注意，而error都需要显示处理。其实在PHP编程中，同样显示处理error也是一种好的变成习惯，虽然会有类似未定义的变量或者undefined的index的数组key会导致不自知的fatal。统统可以通过try
catch来意想不到的未知异常捕获。这里就需要程序员对程序的健壮性强要求，不接受意外的未知的异常。例如，连接超时、数据异常、数据损坏等情况~都需要显示处理。

2、go的设计原则，跟第一条类似。所有代码都保持在可控范围以内，尤其是go的最大的特性，其goroutine的时候，要开发人员保证其起来后，得保证其执行完后停下来。所有的调用方在使用这种并发的便捷性的同时，保证其结果的可靠。

3、go的包管理，是一种类似于php的composer管理工具。经过层层迭代，历经dev时代，到现在的go
mod。个人感觉是一种现代语言包管理的升级。也更符合当下高级语言的更高级。

go mod命令，相当于是将所有依赖的包，当做一个集中处理的模块module，而且最新的go版本，在go
build的时候，也是优先考虑module。减少了每个项目都需要冗余一份的问题。

4、通道channe 是go语言一种独有的类型。而defer 也是go语言一种差别与其他语言的关键字。

chan 通道，make（chan string, 100） 是否带缓冲~ 通道是阻塞的，默认通道是1的队列。如果不消费，会一直阻塞后续写入。

通过 <\- chan 读操作

chan <\- 写操作

一般结合go
关键字使用，go关键字，是go语言能够应付高并发各种计算的原因之一。这里的并发操作，不同于并行。并发操作，是在周期内，尽可能的利用cpu执行力。让使用方，感觉起来是同时执行的并行。

一般在main函数里面，通过go关键字，起来新的线程，来处理计算之类的操作。

defer 延迟关键字
主要是用于main函数退出用于捕获，做相关的后续操作，例如日志，异常，释放资源等~尤其是（释放资源，在open之后，close很重要~）

几个注意点tips：

4.1、多个defer操作，可以看到是一个栈，后进先出的模型。

4.2、defer操作，只是延后执行，如果在defer一瞬间，可以明确获取到所需要的信息，会在defer一开始进入入栈之前，就提前确定赋值。

4.3、在go语言的return中，并不是一个原子操作。是分了两步，第一步，return 赋值， 第二步，真正的ret
返回。在中间可以夹杂处理defer的延迟函数，可能会改变返回值。

4.4、defer只在当前协程生效，goruotine中当前生效，不能捕获其他goroutine中的panic

4.5、os.Exit 系统退出，是不会执行defer函数。os.Exit应该是相当于摧毁了栈信息。

sync.  WaitGroup
关键字，配合go关键字使用。确保go起来的子任务，都完成之后，才会继续执行主流程main接下来的逻辑。tips，sync.WaitGroup
传入的时候，需要以地址传入&wg, 同时子任务接收参数* sync.WaitGroup 类型~ wg.Add()
表明需要起来多少个子任务，每个子任务执行完后，可以defer wg.Done()，表示该任务执行完成，相当于执行wg.Add(-1)

5、go语言的大小写问题，直接替代了php类里面定义的公共变量、私有变量。

go语言中，大写默认是公共变量，都可以访问的。小写是私有变量，只有该包类可以进行访问。
golang中根据首字母的大小写来确定可以访问的权限。无论是方法名、常量、变量名还是结构体的名称，如果首字母大写，则可以被其他的包访问；如果首字母小写，则只能在本包中使用。可以简单的理解成，首字母大写是公有的，首字母小写是私有的
。
